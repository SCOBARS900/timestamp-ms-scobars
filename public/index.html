Timestamp Microservice and
Request Header Microservice

User stories:

    1) User can pass a string as a parameter, and it will check to see whether that string contains either a unix timestamp or a natural language date (example: January 1, 2016 or 1451606400)
    2) If it does, it returns both the Unix timestamp and the natural language form of that date. If it does not it will return null.
    3) User can use the imput field to insert a date then click button "Go" or put directly on the url like the examples bellow
    4) User can click in the button Request Header and it will return the IP adress, language and operating system.
